## 이메일 프로토콜
```
* 프롤로그
  아웃룩(Outlook)이나 윈도우즈메일(Windows Mail)과 같은 이메일 프로그램을 사용하시는 분이라면, 
  설정창에서 SMTP/POP3/IMAP 등의 용어를 접한 적이 있을 것입니다. 
  이러한 용어들은 이메일을 주고 받는 프로토콜의 종류에 해당합니다.

   많은 사람들이 각 프로토콜의 의미를 정확히 알지 못한 채 설정 가이드나 
   다른 사람의 조언을 통해 메일 프로토콜을 설정하곤 합니다. 
   이메일의 기술적인 구조에 대해 간략하게 이해하고, 
   각각의 프로토콜이  의미하는 것이 무엇인지 알아보도록 하겠습니다. 
```

### 이메일 시스템

```
  * 이메일 시스템?
    메일 서버 + 클라이언트로 구성된다. 

  ! 메일 서버
    서버에는 각각의 메일 주소에 대응하는 사서함(MAIL BOX)가 존재합니다.
    사서함을 바탕으로 한 메일 서버 간에 통신 덕분에 원격지에서 시차를 두고 메일을 주고 받을 수 있다.

  ! 메일 클라이언트 
    이메일 클라이언트는 메일 서버의 사서함에 저장된 이메일을 가져와 발신자에게 보여주거나 
    발신자가 작성한 이메일을 서버로 전달하는 역할을 수행합니다.
```

### IMAP

```
* IMAP 란?
  IMAP은 Internet Messaging Access Protocol의 약자로 서버에서 이메일을 읽는 프로토콜

  어떤 사황에서든 동일한 내용을 읽어올 수 있습니다.
```
### POP3
```
* POP3 란?
  Post Office Protocol의 약자로 사용자의 기기로 이메일을 다운로드하여 읽는 프로토콜

  다운로드한 내용은 서버에서 삭제되기 때문에 동일한 기기에서만 이메일 확인이 가능합니다.

+ 서비스 업체에서는 POP3 방식을 많이 제공하고 있으며, 일부 업체에서는 두 가지 방식을 지원한다.
```

### 그렇다면 차이점은 무엇인가?

```
* IMAP는?
  PC나 모바일에서 메일을 확인할 때 서버에 저장된 정보를 가져와서 내용을 보여주고 변경 사항을 동기화합니다.
  그렇기 때문에, 여러 가지의 디바이스에서 접속하여도 항상 동일한 정보를 확인할 수 있습니다.

* BUT POP3는 우리가 사용하는 기기에 정보를 저장합니다.
  서버에 있는 이메일을 열람할 경우 현재 사용하기 있는 기기에 다운로드한 뒤 서버에서는 삭제합니다.
  흔히, 아웃룩 초기 세팅 시 '서버에 복사본 저장' 이라는 설정을 할 수 있는데요

  만약 복사본을 저장하지 않으면 아웃룩에서 주고받는 데이터들이 삭제됩니다.
  왜냐하면 PC 아웃룩에 저장하였고, 복사본을 저장하는 설정도 되어있지 않기 대문에, 
  기본 설정인 다운로드 후 자동 삭제가 이루어집니다.
```

### IMAP VS POP3 장단점 비교

```
* IMAP

  ! 장점
  - 동기화 : 아웃룩에서 보낸 덷이터를 웹 또는 모바일에서 확인할 수 있습니다.
  - 안정적인 환경 : 다운로드 이슈가 발생하지 않아, 안정적인 환경에도 수/발신할 수 있습니다.
  - 편리한 멀티 디바이스 사용 : 언제 어떤 디바이스에서 접속해도 모든 속성이 동기화 되며, 편리하게 사용할 수 있다.

  ! 단점
  - 용량 관리의 어려움 : 서버에 데이터가 지속적으로 쌓이기 때문에, 용량 제한 서비스를 이용할 경우 용량 관리가 어렵다.
  - 비싼 도입 요금 : 모든 내용들을 동기화해야 하기 때문에 더 많은 트래픽이 필요하고 그에 따른 비용이 비싸다.

* POP3

  ! 장점
  - 편리한 용량 관리 : 데이터를 자동적으로 삭제하도록 설정할 수 있어 비교적 편하다.
  - 저렴한 사용 요금 : 거의 모든 서비스 제공 업체가 POP3 환경을 지원하기 때문에 사용 요금이 비싸지 않다.

  ! 단점
  - 보낸 이메일 보존 불가 : 아웃룩에서 보낸 편지는 웹 서버에 저장되지 않기 때문에 아웃룩 외에 다른 곳에서 보낸 데이터를 확인하기 어렵습니다.
  - 기기 관리의 어려움 : 저장된 기기 자체를 관리해야 하기 때문에, 하드 드라이브가 고장 나거나 PC에 바이러스가 생긴 경우 복구하기 어렵습니다.
  - 다운로드 이슈 발생 : 여러 디바이스에서 같은 메일을 계속 다운로드 하다 보면, 중복 다운로드 및 사라짐 이슈 등이 발생할 수 있습니다.
```
